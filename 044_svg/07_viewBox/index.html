Notes

With viewBox, if we make the viewBox smaller, we have zoomed in.

Think about it like this: in the width/height svg area, we are just viewing these pixels of what we're drawing.

For example: in this situation with a width="200" and height="250" the pixels of our drawing which we are viewing are the pixels which would have normally mapped to the width=100 height=125 region of the svg width/height area.

viewBox="0 0 100 125"

From MDN: "The viewBox attribute allows to specify that a given set of graphics stretch to fit a particular container element."

From MDN: "The value of the viewBox attribute is a list of four numbers min-x, min-y, width and height, separated by whitespace and/or a comma, which specify a rectangle in user space which should be mapped to the bounds of the viewport established by the given element, taking into account attribute preserveAspectRatio."

min-x, min-y shift the starting point of all of the zooming. Play with it to see what happens. Try this setting: "0 125 100 125" or this one: "20 0 100 125"


With viewBox, if we make the viewBox larger, we have zoomed out.


viewBox is like meta-viewport in that it creates a zoom effect

<meta name="viewport" content="width=device-width, initial-scale=1">

In meta-viewport, initial-scale allows zooming. "initial-scale=1" says for each css pixel, we will have 1 device-independent pixel. It could "initial-scale=2" (or some other number) in which case for each css pixel, 2 device independent pixels would be used. Thus, in the "initial-scale=2" situation, after 180 css pixels, a screen with 360 device indendent pixels would have all of the pixels used. This would create a zoomed-in effect.
